### Week 3 Goals

- Understand the core concepts of **High Availability** and **Scalability** in a cloud environment.
- Learn about managed database services: **RDS**, **Aurora**, **ElastiCache**.
- Configure and manage **Amazon Route 53** for DNS and traffic routing.
- Study typical architectural models on AWS and practice system design.

### Week 3 Achievements

- Master **High Availability** strategies: Multi-AZ, automated failover, Auto Scaling Group, ELB.
- Expand AWS database knowledge: RDS management, understanding **Aurora** advantages, using **ElastiCache** for speed.
- Master DNS and routing via **Route 53**: hosted zone, routing policy, health check, failover routing.
- Enhance system design thinking: combine **Compute**, **Database**, and **Networking** to build flexible, cost-effective architectures.
